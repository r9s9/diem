# diem — Privacy & Data Transparency

diem is built **private-first**. It is a tool to help *you* remember and report your own
work — not a surveillance tool for managers. This document explains exactly what diem
records, what it never does, and the controls you have. The app shows a condensed version
of this on first run.

## The core promise

- **Everything stays on your laptop.** All activity is stored in an **encrypted database
  on this device only**. There is no diem server. Nothing is uploaded automatically.
- **Encrypted at rest.** The local database is encrypted with SQLCipher. The encryption
  key is sealed to your Windows user account (DPAPI) and held in Windows Credential
  Manager — it is never written to disk in clear text.
- **You own the data.** You can pause tracking, exclude apps/sites, delete any range, or
  wipe everything at any time.

## What diem records

For each stretch of activity on your computer, diem stores a **session**:

| Field | Example | Why |
|-------|---------|-----|
| Active app | `Code.exe`, `Teams.exe` | What you were using |
| Window title | `report.docx — Word` | What you were working on |
| Active / idle / locked | `active` for 23 min | On/off time for the timesheet |
| Browser domain | `github.com` | Which sites, for categorization |
| Browser full URL | `https://github.com/acme/repo/pull/4` | **Only if you opt in** — off by default |
| Open document name | `Q3-budget.xlsx` | Attribute time to a file/project |
| Start / end time | `09:14–09:37` | Build the daily timeline |

It also pulls your **Outlook/Teams calendar** (meetings: subject, time, organizer,
attendees count, online-meeting flag) via Microsoft Graph **after you sign in**, so
meetings appear on your timeline.

## What diem NEVER does

- ❌ No keystroke logging. diem never records *what* you type.
- ❌ No screenshots or screen recording.
- ❌ No microphone, camera, or audio capture.
- ❌ No reading of message/email *contents* — only the app and window title.
- ❌ No background upload of activity. The only data that ever leaves the device is
  described under "Cloud features" below, and only when **you** turn it on.

## AI — local by default

- **Local categorization (default):** activity is sorted into work categories on-device
  using fast rules plus a **local LLM (Ollama)** that runs entirely on your laptop. No
  activity data leaves the machine.
- **Cloud summaries (off by default, opt-in):** you may enable richer weekly/monthly
  narrative summaries generated by the Claude API. When enabled, diem sends **aggregated
  category/time totals** — not raw window titles or URLs — unless you explicitly allow
  detail. This toggle is clearly labeled and off until you choose it.

## Your controls

- **Pause / Incognito** — stop tracking instantly from the tray.
- **Exclusion lists** — name apps or domains (e.g. your bank, personal email) that diem
  must never record.
- **Domain-only mode** — record `github.com` instead of the full URL (default).
- **Delete & export** — remove any time range; export your own summary to paste into a
  timesheet. Wipe all data in one click.

## A note for employers deploying diem

Employee-monitoring software carries **notice and consent obligations that vary by
country, state, and works-council agreement**. diem's private-by-design model is intended
to be employee-friendly, but deploying it does not, by itself, satisfy your legal
obligations. Consult HR/legal and inform employees before rollout. This file is a
transparency aid, not legal advice.
